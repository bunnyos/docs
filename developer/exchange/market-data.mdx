The bunnyDEX REST API exposes the read side of the marketplace: the list of bunnies, per-key price history, the global trade tape, and a caller's profit-and-loss ledger. These endpoints power dashboards, bots, and analytics without ever touching a private key.

<Note>
**Reads, not trades.** Buying and selling bunny keys happens **on-chain**, against the bonding-curve contract on **Base mainnet (chainId `8453`)**, not through this REST API. The endpoints on this page are read-only market and account data. See [Authentication](/developer/exchange/authentication) for how to mint a JWT and [bunnyDEX API overview](/developer/exchange/overview) for the full picture.
</Note>

All requests target the hosted base URL:

```
https://api.bunnyos.ai
```

## Authentication for reads

Reads accept an optional JWT or `x-api-key`; P&L endpoints require a JWT because they describe *your* positions. See [Authentication](/developer/exchange/authentication) for how to mint either credential.

| Endpoint | Method | Auth |
| --- | --- | --- |
| `/exchange/bunnies` | `GET` | Optional JWT |
| `/exchange/bunnies/{id}` | `GET` | Optional JWT |
| `/exchange/bunnies/{id}/candles` | `GET` | Optional JWT |
| `/exchange/bunnies/{id}/transactions` | `GET` | Optional JWT |
| `/exchange/bunnies/{id}/pnl` | `GET` | JWT required |
| `/exchange/portfolio/pnl` | `GET` | JWT required |

## Listing bunnies

A bunny is a tradable source of on-chain action recommendations: a person, team, algorithm, or AI whose keys trade on the bonding curve. Each bunny is identified by an on-chain id (its token id) and may also have a bunnyOS-hosted profile.

| Endpoint | Method | Auth |
| --- | --- | --- |
| `/exchange/bunnies` | `GET` | Optional JWT |
| `/exchange/bunnies/{id}` | `GET` | Optional JWT |

`GET /exchange/bunnies` returns every bunny, and `GET /exchange/bunnies/{id}` returns a single bunny. The list response wraps entries under `bunnies`, and each entry carries `bunnyId` for identity plus profile fields (`name`, `description`, `photoUrl`, and more). For the full profile schema and owner-only writes, see [Profile](/developer/exchange/profile).

```bash
curl https://api.bunnyos.ai/exchange/bunnies

curl https://api.bunnyos.ai/exchange/bunnies/42
```

<Note>
A bunny can exist **on-chain** (its keys trade) before anyone creates a bunnyOS profile for it. In that case the profile fields are empty or `null` (no `name`), and `GET /exchange/bunnies/{id}` returns **404** when bunnyOS has no profile at all for that id. The absence of a profile does **not** mean the bunny can't be traded, price and trade data still exist on-chain.
</Note>

## Price candles (OHLC)

`GET /exchange/bunnies/{id}/candles` returns real OHLC price history for a bunny's keys.

| Endpoint | Method | Auth |
| --- | --- | --- |
| `/exchange/bunnies/{id}/candles` | `GET` | Optional JWT |

### Query parameters

| Param | Required | Notes |
| --- | --- | --- |
| `interval` | No | One of `1m`, `5m`, `15m`, `1h`, `4h`, `1d`, `1w`. Defaults to `1h`. An unrecognized value is a `400`. |
| `limit` | No | `1`-`1000`. Out-of-range values are a `400`. |

### Response

```json
{
  "bunnyId": "42",
  "interval": "1h",
  "candles": [
    {
      "bucketStart": "2026-06-16T12:00:00Z",
      "open": "1010000000000000",
      "high": "1180000000000000",
      "low": "1000000000000000",
      "close": "1150000000000000",
      "volumeKeys": "37",
      "volumeValue": "42000000000000000",
      "tradeCount": 12
    }
  ]
}
```

```bash
curl "https://api.bunnyos.ai/exchange/bunnies/42/candles?interval=1h&limit=200"
```

<Warning>
`open`, `high`, `low`, `close`, and `volumeValue` are **raw wei decimal strings** in the smallest unit of the `$OS` token (see [token units](#bonding-curve-context--token-units) below); convert them with `formatUnits` using the OS token's decimals. `volumeKeys` is a count of keys. `interval` is optional and defaults to `1h`; an unrecognized value is a `400`.
</Warning>

## Trade history (the trade tape)

`GET /exchange/bunnies/{id}/transactions` returns the **global** trade tape for a bunny, every trade by every trader, newest first.

| Endpoint | Method | Auth |
| --- | --- | --- |
| `/exchange/bunnies/{id}/transactions` | `GET` | Optional JWT |

### Query parameters

| Param | Required | Notes |
| --- | --- | --- |
| `limit` | No | `1`-`1000`. |
| `cursor` | No | Opaque pagination cursor from a prior response. |

### Response

```json
{
  "bunnyId": "42",
  "transactions": [
    {
      "side": "buy",
      "amount": "3",
      "pricePerKey": "1150000000000000",
      "total": "3450000000000000",
      "trader": "0xabc...",
      "timestamp": "2026-06-16T12:31:04Z"
    }
  ],
  "nextCursor": "eyJ...",
  "hasMore": true
}
```

```bash
# First page
curl "https://api.bunnyos.ai/exchange/bunnies/42/transactions?limit=100"

# Next page
curl "https://api.bunnyos.ai/exchange/bunnies/42/transactions?limit=100&cursor=eyJ..."
```

<Steps>
  <Step title="Request the first page">
    Call the endpoint with a `limit`. Read `transactions` (newest first).
  </Step>
  <Step title="Check hasMore">
    If `hasMore` is `true` and `nextCursor` is non-null, more pages remain.
  </Step>
  <Step title="Pass the cursor forward">
    Send `nextCursor` as the `cursor` query param on the next call. Stop when `hasMore` is `false` or `nextCursor` is `null`.
  </Step>
</Steps>

<Note>
`pricePerKey` and `total` are **raw wei decimal strings** in the smallest unit of the `$OS` token; `amount` is a count of keys. `side` distinguishes buys from sells. This is the public tape, so it works anonymously.
</Note>

## Profit & loss (P&L)

P&L endpoints report your realized and unrealized gains against your **cost basis**. They are the official ledger authority for a connected account and therefore **require a JWT**.

| Endpoint | Method | Auth |
| --- | --- | --- |
| `/exchange/bunnies/{id}/pnl` | `GET` | JWT required |
| `/exchange/portfolio/pnl` | `GET` | JWT required |

### Per-bunny P&L

```bash
curl https://api.bunnyos.ai/exchange/bunnies/42/pnl \
  -H "Authorization: Bearer <jwt>"
```

```json
{
  "bunnyId": "42",
  "balance": "5",
  "costBasis": "5000000000000000",
  "realizedPnl": "200000000000000",
  "unrealizedPnl": "1500000000000000",
  "totalPnl": "1700000000000000"
}
```

This endpoint requires a valid JWT. Without one it returns `401` (the credential is missing or expired); it does not fall back to an anonymous ledger.

### Portfolio P&L

`GET /exchange/portfolio/pnl` aggregates across every bunny you hold and adds `totals`.

```bash
curl https://api.bunnyos.ai/exchange/portfolio/pnl \
  -H "Authorization: Bearer <jwt>"
```

```json
{
  "totals": {
    "costBasis": "5000000000000000",
    "realizedPnl": "200000000000000",
    "unrealizedPnl": "1500000000000000",
    "totalPnl": "1700000000000000"
  },
  "bunnies": [
    {
      "bunnyId": "42",
      "balance": "5",
      "costBasis": "5000000000000000",
      "realizedPnl": "200000000000000",
      "unrealizedPnl": "1500000000000000",
      "totalPnl": "1700000000000000"
    }
  ]
}
```

### Cost basis vs invested / returned

The P&L ledger is built around **cost basis**, which is a more precise concept than a naive "total invested minus total returned":

- **`costBasis`** is the accounting cost of the keys you **currently still hold**. As you sell, the matching portion of cost is removed from the basis and the difference between sale proceeds and that removed cost is booked into `realizedPnl`. So `costBasis` is *not* the same as everything you ever put in; it shrinks as you exit positions.
- **`realizedPnl`** is locked-in profit or loss from keys you have already sold.
- **`unrealizedPnl`** is the mark-to-market gain or loss on keys you still hold, measured against their cost basis at the current curve price.
- **`totalPnl`** combines both: `realizedPnl + unrealizedPnl`.

<Tip>
If you only track gross invested and gross returned, you will misattribute gains across partial exits. Use `costBasis` together with `realizedPnl` and `unrealizedPnl` to reconcile a position correctly. Every value field except `balance` is a **raw wei decimal string** in the smallest unit of the `$OS` token; `balance` is a count of keys.
</Tip>

<Warning>
**No JWT, no P&L.** P&L has no anonymous form. Without a valid JWT these endpoints return **`401`** rather than a ledger, so calling code should prompt the user to sign in rather than rendering zeroed numbers as if they were real. (Optional-auth reads instead retry anonymously; see [Authentication](/developer/exchange/authentication).)
</Warning>

## Bonding-curve context & token units

Key prices on bunnyOS are set by a **bonding curve** on Base: each successive key costs more than the last along a deterministic curve, so price is a function of supply. Trades execute against that contract on-chain; the REST API observes the resulting state. That is why:

- **Candles** describe per-key price movement as the curve advances and retreats with buys and sells.
- **Transactions** are the realized fills against the curve (`pricePerKey`, `amount`, `total`, `side`).
- **P&L** values your holdings at the current curve price (`unrealizedPnl`) versus what you paid (`costBasis`).

Two unit conventions matter throughout:

1. **`amount` / `balance` / `volumeKeys`** are **counts of keys**: small integers.
2. **All price and value fields** (`open`/`high`/`low`/`close`, `volumeValue`, `pricePerKey`, `total`, `costBasis`, `realizedPnl`, `unrealizedPnl`, `totalPnl`) are **raw wei decimal strings** in the smallest unit of the `$OS` token (for example `"1150000000000000"`), not already-converted token amounts. To get a human `$OS` amount, run them through `formatUnits` (or divide by `10 ** decimals`) using the OS token's on-chain `decimals` (read from the `$OS` ERC-20 token contract on Base).

These are strings rather than JSON numbers to preserve precision. Convert them with a decimal- or BigInt-aware library when you need exact arithmetic; the values are too large to read safely as a plain JavaScript `Number`.

### Worked example: read a bunny end to end

```bash
# 1. Resolve the bunny's profile (may 404 if no bunnyOS profile yet)
curl https://api.bunnyos.ai/exchange/bunnies/42

# 2. Pull hourly candles for charting
curl "https://api.bunnyos.ai/exchange/bunnies/42/candles?interval=1h&limit=168"

# 3. Pull the latest trades
curl "https://api.bunnyos.ai/exchange/bunnies/42/transactions?limit=50"

# 4. (Authenticated) read your position in this bunny
curl https://api.bunnyos.ai/exchange/bunnies/42/pnl \
  -H "Authorization: Bearer <jwt>"
```
